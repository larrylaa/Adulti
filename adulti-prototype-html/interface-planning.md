# Product Interface Planning

## Screen Architecture
- home.html: Financial snapshot + next action focus.
- roadmap.html: Prioritized roadmap with completion and generic flow view.
- guide.html: Guide index with categorized topics and quick access to detail.
- guide-detail.html: Long-form educational content for one roadmap concept.
- debt-editor.html: Focused debt entry/editing for card and loan balances.
- profile.html: User identity, milestones, and account status.
- settings.html: App preferences, privacy controls, and account actions.

## Information Hierarchy Principles
- Top-level data first, actions second, detail third.
- Keep mobile sections short and scannable.
- Highlight progress and current stage to improve continuity.
- Keep controls close to related data to reduce navigation overhead.
